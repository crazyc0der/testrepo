Работа с feature

  • Начало разработки фичи:

    git flow feature start MYFEATURE

    Cоздаёт новую ветку фичи, основанную на ветке "develop", и переключается на неё.

  • Завершение фичи:

    git flow feature finish MYFEATURE

    Cоздаёт новую ветку фичи, основанную на ветке "develop", и переключается на неё.

  • Публикация фичи:

    git flow feature publish MYFEATURE

    Публикует фичу на удаленном сервере.

  • Получение опубликованной фичи:

    git flow feature pull origin MYFEATURE

    Публикует фичу на удаленном сервере.

    Можно отслеживать фичу в репозитории origin с помощью команды:

    git flow feature track MYFEATURE

Работа с релизами

  • Начало релиза:

    git flow release start RELEASE

    Cоздаёт ветку релиза, ответвляя от ветки develop.

    Далее нужно опубликовать ветку релиза, чтобы другие могли делать в нее коммиты.

    git flow release publish RELEASE

    Отслеживать релиз можно с помощью команды:

    git flow release track RELEASE

  • Завершение релиза:

    git flow release finish RELEASE

    Ветка релиза сливается в master

    Релиз помечается тегом с именем, равным его имени.

    Ветка релиза сливается обратно в ветку develop.

    Ветка релиза удаляется.

    Далее нужно отправить изменения:

    git push --tags
